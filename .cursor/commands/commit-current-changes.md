# Commit current changes

## Overview
Write a well-structured, very concise commit message and commit it

## Steps
1. **Check currently staged changes**
   - Check `git diff --cached`

2. **Write commit message**
    - Keep commit title concise
    - Summarize changes clearly
    - Include context and motivation
    - List any breaking changes
    - Do not write long phrases, keep things short
    - Do not devote multiple bullets to test-related changes, at most, use a single bullet point for describing tests

3. **Git commit**
    - Commit the new message
